日志文件大小管理机制

在日志系统中，当日志文件大小达到设定的大小限制时，会触发一系列处理流程以确保日志系统正常运行并合理利用磁盘空间。

1. 文件大小检测

系统会在写入每条日志消息前检查当前日志文件大小是否已达到设定的最大值（默认为10MB）：

if (File.Exists(_currentFileName) && new FileInfo(_currentFileName).Length >= MaxFileSize) { CloseFile(); }

2. 创建新文件

当检测到当前文件需要关闭时（因为大小超限或日期变更），系统会：

  1. 关闭当前日志文件
  2. 创建一个新的日志文件

新文件命名规则

新文件基于以下格式命名：

基础文件名_日期_序号.扩展名

例如：应用日志_20230516_1.log

系统会自动递增序号，直到找到不存在或未超过大小限制的文件名：

string dateStr = localDate.ToString("yyyyMMdd"); string baseFileNameWithDate = $"{BaseFileName}_{dateStr}"; int sequenceNumber = 1; string fileName; do { fileName = Path.Combine(directory, $"{baseFileNameWithDate}_{sequenceNumber}{FileExtension}"); sequenceNumber++; } while (File.Exists(fileName) && new FileInfo(fileName).Length >= MaxFileSize);

3. 文件压缩（可选）

如果启用了压缩功能，系统会在关闭日志文件后对其进行压缩：

if (enabledCompression && FileExtension.EndsWith(".gz", StringComparison.OrdinalIgnoreCase) && File.Exists(currentFileName)) { CompressFileAsync(currentFileName); }
注意： 压缩过程是异步进行的，不会阻塞主线程，确保日志记录操作可以继续进行。

4. 旧文件清理

在创建新文件后，系统会触发清理任务，该任务基于两个主要规则：

4.1 文件数量限制

如果日志文件数量超过设定的最大数量（默认为10个），系统会删除最旧的文件：

var allFiles = Directory.GetFiles(directory, $"{BaseFileName}_*{FileExtension}") .Select(f => new FileInfo(f)) .OrderByDescending(f => f.LastWriteTime) .ToArray(); if (allFiles.Length > MaxNumFiles) { for (int i = MaxNumFiles; i < allFiles.Length; i++) { File.Delete(allFiles[i].FullName); } }

4.2 文件保留期限

系统会删除超过设定保留天数（默认为30天）的旧文件：

DateTime oldestAllowed = DateTime.Now.AddDays(-MaxDaysOld); var oldFiles = Directory.GetFiles(directory, $"{BaseFileName}_*{FileExtension}") .Select(f => new FileInfo(f)) .Where(f => f.LastWriteTime < oldestAllowed) .ToArray(); foreach (var file in oldFiles) { File.Delete(file.FullName); }

总结

当日志文件达到设定大小后，系统会执行以下操作：

  1. 关闭当前文件
  2. 根据命名规则创建新文件
  3. 如果启用了压缩，异步压缩旧文件
  4. 触发旧文件清理任务，根据数量和时间限制删除不需要的文件

这种机制确保了：

  • 日志文件大小可控，便于查看和传输
  • 通过压缩功能节省磁盘空间
  • 自动清理旧文件，避免磁盘空间耗尽
  • 整个过程对应用程序透明，不影响正常日志记录
